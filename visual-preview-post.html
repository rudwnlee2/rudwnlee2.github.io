다시 읽기 쉬운 코드를 남기는 기준

나중에 다시 보는 사람을 위해 이름, 함수 경계, 기록을 어떻게 남길지 정리한 짧은 글입니다.

2026.06.29조회 128댓글 3

코드는 실행보다 오래 남는다

기능을 끝내는 순간에는 코드가 잘 동작하는지만 보이기 쉽다. 하지만 시간이 지나 다시 그 코드를 열어보면, 진짜 비용은 실행 여부보다 이해 가능성에서 나온다.

이름은 작은 문서다

변수와 함수 이름은 가장 자주 읽히는 문서다. 그래서 이름에는 구현 방법보다 역할이 드러나는 편이 좋다.

const expiredSubscriptions = subscriptions.filter((subscription) => {
    return subscription.expiresAt < today;
});

나중에 고칠 사람을 위한 체크리스트

  • 이름은 결과와 역할을 말한다.
  • 함수는 한 가지 판단만 담당한다.
  • 예외 상황은 조용히 숨기지 않는다.
