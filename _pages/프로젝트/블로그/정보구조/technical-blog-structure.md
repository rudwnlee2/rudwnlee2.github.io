## 처음에는 상단을 좁게 둔다

기술 블로그는 글이 늘어날수록 분류가 복잡해지기 쉽다. 그래서 상단 메뉴는 처음부터 모든 주제를 나열하지 않고, 전체와 Backend, 프로젝트처럼 가장 자주 쓰는 입구만 둔다.

이 방식은 새 글을 쓸 때도 부담을 줄인다. 글이 어느 큰 흐름에 속하는지만 먼저 정하고, 세부 분류는 나중에 필요해질 때 좌측 카테고리에서 확장하면 된다.

## 좌측 카테고리는 위치를 알려준다

글 상세 화면에서 독자는 지금 읽는 글이 어느 맥락에 있는지 빠르게 알고 싶다. 상단 메뉴가 큰 길잡이라면 좌측 카테고리는 현재 위치를 알려주는 지도에 가깝다.

2뎁스는 기본으로 충분하고, 프로젝트가 길어지거나 Backend 안에서 Spring, Database처럼 주제가 쌓일 때만 3뎁스를 연다. 깊이를 많이 만드는 것보다 다시 찾기 쉬운 이름을 붙이는 편이 중요하다.

## 글 상세에서 먼저 보여야 할 것

글 상세 화면은 제목, 설명, 날짜, 조회수, 댓글 수, 태그, 목차, 관련 글이 한 흐름 안에서 보여야 한다. 특히 코드블럭 복사 버튼과 검색, 다크모드는 기술 블로그에서 반복 사용되는 기능이므로 작은 화면에서도 유지되어야 한다.

```js
const navigationPrinciple = {
    topLevel: ["전체", "Backend", "프로젝트"],
    sidebarDepth: "2뎁스를 기본으로, 필요할 때 3뎁스",
    postDetail: ["제목", "설명", "메타", "목차", "관련 글"]
};
```

## 운영하면서 조정할 기준

- 상단 메뉴는 세 개 안팎으로 유지한다.
- 좌측 카테고리는 현재 위치를 보여주는 용도로 둔다.
- 태그는 검색과 관련 글 연결에 쓴다.
- 코드블럭 복사, 다크모드, 모바일 메뉴는 항상 확인한다.
